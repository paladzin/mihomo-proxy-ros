# Руководство по участию в проекте

Спасибо за интерес к **mihomo-proxy-ros** ❤️  
Любые предложения, исправления и улучшения приветствуются!

---

## 🧩 Как предложить изменения

1. **Создайте форк** репозитория на GitHub.  
2. **Создайте отдельную ветку** для своих изменений:

    ```
    git checkout -b fix/readme-typo
    ```

3. **Внесите правки** и убедитесь, что всё корректно собирается.  
4. **Создайте Pull Request (PR)** в ветку `main`, описав цель изменений.

---

## 💡 Рекомендации

- Старайтесь оформлять коммиты в читаемом виде:

    ```
    feat: добавлен ENV для настройки mihomo-proxy-ros
    fix: исправлен баг при установке на ARM64
    docs: обновлена инструкция MikroTik
    ```

- Перед созданием Pull Request убедитесь, что:
  - Dockerfile успешно собирается;
  - скрипт для RouterOS работает без ошибок;
  - документация (README и комментарии в коде) обновлены при необходимости.

---

## 🐞 Ошибки и предложения

Если вы нашли баг или хотите предложить улучшение:

1. Проверьте, нет ли уже похожего issue.  
2. Если нет — создайте **новое issue**, подробно описав:
   - платформу (MikroTik, архитектуру, версию Docker);
   - шаги для воспроизведения;
   - ожидаемое и фактическое поведение;
   - логи (если есть).

---

## ⚙️ Среда разработки

Проект собирается через **Docker** и поддерживает мультиархитектуру:

- **ARM / ARM64 / AMD64v1–v3**
- Базируется на **Mihomo**
- Поддерживает установку через **RouterOS script**
